Hotfix release — Apple Silicon (ARM) macOS globe rendering fix and stability improvements.

**CORS Fix for Local Renderer Server (Root Cause)**
- Added CORS headers (`Access-Control-Allow-Origin: *`) to the local HTTP server (`startRendererServer`)
- When migrating from `app://` protocol (v1.0.0) to `http://127.0.0.1` (v1.0.1+), CORS headers were missing
- Cesium tile images loaded with `crossOrigin` attribute were marked as "tainted" by the browser
- WebGL `texImage2D` silently failed on tainted images → black globe on macOS (especially Apple Silicon)
- Added OPTIONS preflight handler for proper CORS compliance

**Apple Silicon Shader Workaround**
- Skip CesiumJS imagery layer shader parameters (brightness, contrast, saturation, gamma)
  on Apple Silicon Macs to work around ANGLE/Metal WebGL bug
  (https://github.com/CesiumGS/cesium/issues/11251)

**Server MIME Type Fix**
- Added `.xml` → `application/xml` MIME type to local renderer HTTP server
- Fixes Cesium NaturalEarthII `tilemapresource.xml` being served as `application/octet-stream`

**Startup Crash Fix**
- Fixed `offline-cache-service.ts` crash when `app.getPath('userData')` is called before `app.whenReady()`
- Cache directory path is now lazily initialized on first `init()` call

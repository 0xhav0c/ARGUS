Hotfix release — Apple Silicon macOS globe rendering, fullscreen crash, and stability improvements.

**Apple Silicon Packaged Build Fix (Root Cause)**
- Cesium tile assets (jpg, xml, js, wasm) are now excluded from asar archive via `asarUnpack`
- In packaged DMG, `createReadStream` inside asar is unreliable for binary files on Apple Silicon
- Cesium requests are redirected to `app.asar.unpacked/` path at the HTTP server level
- This was the actual root cause: dev build worked fine, only packaged DMG had black globe

**CORS Headers for Local Renderer Server**
- Added `Access-Control-Allow-Origin: *` to all HTTP server responses
- Added OPTIONS preflight handler for proper CORS compliance
- Without CORS, Cesium tile images loaded with `crossOrigin` were marked as tainted by WebGL

**Apple Silicon Shader Workaround**
- Imagery layer shader parameters (brightness, contrast, saturation, gamma) conditionally
  skipped on Apple Silicon to avoid ANGLE/Metal WebGL bug
  (https://github.com/CesiumGS/cesium/issues/11251)

**Fullscreen Mode Crash Fix (Windows)**
- Fixed React Error #300 ("Rendered fewer hooks than expected") when clicking FULLSCREEN button
- `useSettingsStore` hook was placed after a conditional early return, violating React's rules of hooks
- Hook call moved above the `mapFullscreen` conditional return

**Other Fixes**
- Added `.xml` MIME type to local renderer server for Cesium `tilemapresource.xml`
- Lazy-init `offline-cache-service.ts` to prevent `app.getPath()` crash before `app.whenReady()`
